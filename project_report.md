# SYNQ 前端期末/期中專案說明書 (Q&A 導向)

這份文件旨在協助你回覆老師關於「這個網站是如何製作的」、「排版邏輯是什麼」等技術與設計問題。

---

## 1. 核心技術選型 (Tech Stack)

**Q: 這個網站使用了哪些技術？有使用任何框架嗎？**
**A:** 我們這個專案完全遵循「關注點分離 (Separation of Concerns)」的原則：
- **結構**：純 HTML5 (語意化標籤，如 `<header>`, `<main>`, `<section>`)
- **樣式**：Tailwind CSS (透過 CDN 引入原子化類別) 加上手寫的 Vanilla CSS (用於全局變數、毛玻璃效能優化與特製的 keyframes 動畫)。
- **邏輯**：原生 JavaScript (Vanilla JS)，不依賴 React 或 Vue 等前端框架。
- **第三方套件**：僅引入預先寫好的 AOS.js (Animate On Scroll) 來處理滾動時的淡入與滑進動畫，確保網站輕量且效能純粹。

之所以不使用大型前端框架，是為了保留最純粹的靜態部署能力，只需點開 `index.html` 即可在瀏覽器完美運行，非常適合展示前端最基礎但也最重要的網頁切版能力。

## 2. 網格排版邏輯 (Bento Grid Layout)

**Q: 首頁中間那個產品展示區塊 (Bento Grid) 的排版是怎麼做的？這在 RWD (響應式設計) 上怎麼適應手機版？**
**A:** 那個區塊使用了現代很流行的 ** CSS Grid Box (Bento Grid 設計語言)**：
- **桌面版 (Desktop)**：利用 CSS Grid (`grid-cols-3`) 切成三等份。
  - 左側的「AI Smart Keyboard」利用 `grid-column: span 2` 與 `grid-row: span 2` 向右、向下方跨越兩個格子，形成一個具備視覺焦點的「大卡片 (bento-large)」。
  - 右邊兩個產品 (Macro Deck, Neural Synth) 則是標準的一格大 (`bento-medium`)，自然堆疊在右側剩餘的 Grid 空間內。
- **手機版 (Mobile)**：由於手機螢幕較窄，我們透過 Media Query 設定（Tailwind 的 `md:` 前綴），將 `grid-cols-3` 退化為 `grid-cols-1`。同時把 `.bento-large` 的跨越屬性（`span 2`）限制在桌面版以上才會生效。這樣一來，在手機上所有產品卡片都會順利切換成「上下堆疊」的瀑布流佈局，避免因為強迫跨越兩格導致版面破版或互相重疊。

## 3. 視覺與設計美學 (UI/UX Design)

**Q: 網站整體的毛玻璃背景、霓虹發光感跟暗色模式很特別，這些 CSS 效果具體是怎麼刻出來的？**
**A:** 我們的視覺核心建立在幾種 CSS 進階技巧的疊加：
1. **Glassmorphism (毛玻璃)**：我們使用了 `backdrop-filter: blur(12px)` 來實現視窗模糊，並配上極低透明度的背景白 (`rgba(255,255,255,0.035)`) 與 1px 的半透明邊框。這讓元素的層次感能凸顯在後方深色的網頁背景之上，同時保留通透感。
2. **Neon Glow (霓虹光暈)**：我們在 `:root` 設定了主視覺的紫色 (`#a855f7`) 與冰藍色 (`#7dd3fc`)。光暈效果是利用多層 `box-shadow`（內層亮、外層散的雙層陰影）加上 `@keyframes pulse`，讓卡片上的光暈能夠以呼吸的方式漸進閃爍。
3. **Noise Texture (背景噪點)**：為了避免純黑背景過於死板，我們利用 SVG 的 `feTurbulence` (分形噪聲濾波器) 渲染出了一個噪點圖，並透過 `body::before` 以 `opacity: 0.45` 鋪在最下層，創造出具有工業感的高級硬體材質視覺體驗。

## 4. 目錄結構與進階細節

**Q: 可以講講你為什麼特別設立一個 js 和 css 資料夾，而不是直接全寫在 HTML 裡嗎？**
**A:** 基於軟體工程的**關注點分離、高內聚低耦合**觀念，將這些邏輯切開可以：
1. **提高可維護性**：如果我們要增加新的頁面 (如 `keyboard.html`)，它們可以直接共用同一支 `style.css` 與 `script.js`，避免每次修改全局按鈕顏色都要進入每一頁 HTML 修改。
2. **優化瀏覽器快取 (Caching)**：當使用者第一次載入 `style.css` 與 `script.js` 後，瀏覽器就會把它們快取起來。未來換到其他分頁時，只需要純載入 HTML，能大幅加快載入速度。
