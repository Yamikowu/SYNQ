## 一、技術架構類

### Q1: 用了什麼技術？為什麼選擇這些技術？

**回答：**
```
技術棧：
- HTML5 語義化標籤
- CSS3（自訂變數、動畫、Grid、Flexbox）
- Tailwind CSS CDN（原子化框架）
- 原生 JavaScript（ES6+）
- AOS 動畫庫

選擇原因：
1. Tailwind CDN：快速開發，無需建置工具
2. 原生 JS：無框架依賴，載入速度快
3. AOS：輕量級滾動動畫，提升 UX
```

---

### Q2: 程式碼如何組織？為什麼這樣分離？

**回答：**
```
架構原則：關注點分離 (Separation of Concerns)

目錄結構：
├── css/
│   ├── style.css          # 全域樣式、設計系統變數
│   └── pages/             # 各頁面專用樣式
├── js/
│   ├── script.js          # 核心互動邏輯
│   ├── layout.js          # 共用版面載入
│   ├── layout-templates.js # HTML 模板
│   └── pages/             # 各頁面專用腳本
├── assets/img/            # 圖片資源
└── *.html                 # 各頁面

優點：
1. 易於維護：修改導覽列只需改一個檔案
2. 可重用性：共用元件跨頁面使用
3. 快取優化：CSS/JS 分離可被瀏覽器快取
```

## 二、HTML/CSS 相關

### Q3: 為什麼使用語義化標籤？

**回答：**
```html
使用的語義化標籤：
- <header>：導覽列區域
- <main>：主要內容
- <section>：各個內容區塊
- <footer>：頁尾資訊
- <nav>：導覽選單

好處：
1. SEO 友善：搜尋引擎更容易理解內容結構
2. 無障礙：螢幕閱讀器能正確解讀頁面
3. 可讀性：開發者更容易理解程式碼結構
4. 維護性：修改時更容易找到對應區塊
```

### Q4: 響應式設計如何實現？

**回答：**
```
實現方式：
1. Tailwind 斷點系統：
   - sm: 640px 以上
   - md: 768px 以上
   - lg: 1024px 以上
   - xl: 1280px 以上

2. 彈性佈局：
   - Flexbox：導覽列、按鈕組
   - CSS Grid：產品卡片 Bento 佈局

3. 相對單位：
   - rem/em：字體大小
   - %/vw/vh：寬度高度
   - max-width：限制最大寬度

4. 媒體查詢：
   - 漢堡選單在 lg 以下顯示
   - 網格欄數隨螢幕調整
```

**回答：**

2. 可維護性：改一個變數全站生效

```

## 三、JavaScript 相關

### Q6: JavaScript 如何組織的？用了什麼設計模式？

**回答：**
```
組織方式：
1. 模組化：使用 IIFE（立即執行函式）避免全域污染
2. 事件驅動：DOMContentLoaded 初始化
3. 模板分離：HTML 模板獨立在 layout-templates.js

設計模式：
- 單例模式：AOS 初始化只執行一次
- 觀察者模式：CustomEvent 通知版面更新
- 策略模式：不同頁面載入不同腳本

程式碼範例：
```javascript
// IIFE 避免全域污染
(function initSharedLayout() {
  // 私有函式
  function getPageKey() { ... }
  
  // 公開介面
  window.SYNQ_LAYOUT = { ... };
})();
```

### Q7: AOS 動畫如何實現？為什麼選擇這個庫？

**回答：**
```
實現方式：
1. 載入 AOS CSS 和 JS（CDN）
2. 在 HTML 元素添加 data-aos 屬性
3. 初始化 AOS.init() 配置

HTML 範例：
```html
<div data-aos="fade-up" 
     data-aos-duration="800" 
     data-aos-delay="100">
  內容
</div>
```

選擇原因：
1. 輕量級：gzip 後僅 3KB
2. 易用性：只需添加 data 屬性
3. 效能佳：使用 Intersection Observer API
4. 無依賴：純 JS 實現，無需 jQuery
```

## 四、功能實現

### Q8: 導覽列的 active 狀態如何實現？

**回答：**
```javascript
// 1. 根據當前頁面路徑判斷
function getPageKey() {
  const file = window.location.pathname.split('/').pop();
  if (file === 'controller.html') return 'controller';
  if (file === 'macro.html') return 'macro';
  // ...
}

// 2. 找到對應的導覽連結並添加樣式
function applyActiveNav() {
  const pageKey = getPageKey();
  const links = document.querySelectorAll(`[data-nav="${pageKey}"]`);
  links.forEach(link => {
    link.classList.add('text-ice');
  });
}

優點：
- 純前端實現，無需後端
- 支援多個相同連結（桌面版 + 行動版）
```

### Q9: 滾動進度條如何實現？

**回答：**
```javascript
// 監聽滾動事件
window.addEventListener('scroll', () => {
  // 計算滾動百分比
  const scrollTop = window.scrollY;
  const docHeight = document.documentElement.scrollHeight - window.innerHeight;
  const scrollPercent = (scrollTop / docHeight) * 100;
  
  // 更新進度條寬度
  document.getElementById('scroll-bar').style.width = scrollPercent + '%';
});

CSS：
```css
#scroll-bar {
  position: fixed;
  top: 0;
  left: 0;
  height: 3px;
  background: linear-gradient(90deg, var(--color-ice), var(--color-neon));
  z-index: 9999;
  transition: width 0.1s;
}
```

## 五、設計與 UX

### Q10: 配色方案如何選擇？有什麼設計考量？

**回答：**
```
配色方案：
- 主背景：#070b14（深藍黑，科技感）
- 次背景：#0e1628（層次分明）
- 主色：#5ee6ff（電藍，AI/科技意象）
- 強調色：#a855f7（霓虹紫，未來感）

設計考量：
1. 品牌一致性：科技硬體品牌常用深色系
2. 對比度：確保文字可讀性（WCAG 標準）
3. 情感傳達：藍色代表信任，紫色代表創新
4. 視覺層次：使用不同深淺創造深度感
```

### Q11: 如何確保網站的可訪問性（Accessibility）？

**回答：**
```
實作項目：
1. 語義化標籤：使用 header, main, nav, footer
2. ARIA 屬性：
   - aria-label：按鈕說明
   - aria-expanded：選單狀態
   - aria-hidden：裝飾性元素
3. 鍵盤導覽：所有互動元素可 Tab 聚焦
4. 對比度：文字與背景對比度 ≥ 4.5:1
5. 圖片替代文字：所有 img 有 alt 屬性

範例：
```html
<button aria-label="開啟選單" aria-expanded="false">
  <span class="sr-only">選單</span>
</button>
```

## 六、效能優化

### Q12: 有做哪些效能優化？

**回答：**
```
優化措施：
1. 圖片優化：
   - 使用適當格式（PNG 用於圖標，JPG 用於照片）
   - 響應式圖片（不同螢幕載入不同尺寸）

2. 載入策略：
   - CSS 放 head（避免 FOUC）
   - JS 放 body 底部或使用 async/defer
   - AOS 動態載入（需要時才載入）

3. 快取策略：
   - 靜態資源分離（CSS/JS 獨立檔案）
   - 使用 CDN（Tailwind, AOS, Google Fonts）

4. 程式碼優化：
   - 避免重複 DOM 查詢
   - 使用事件委派
   - 防抖/節流（滾動事件）
```
